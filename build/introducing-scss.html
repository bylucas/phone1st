Introducing Scss

Start using scss for smart, efficient and quicker stylesheets. Forget about writing long winded css. The scss basics are easy to pick-up and once you start using scss you’ll keep using it. After reading this article, you will find the Sass website useful.

What is Scss?

Scss enables the developer to write quicker and more efficient css. The idea is to use partial stylesheets for different sections i.e. typography, forms, images, etc and have a main stylesheet collect these partial stylesheets. Then output the main scss stylesheet, via a pre-processor, (see below), to a css file. This together with variables and mixins, (see below), make writing css more efficient. It’s not difficult to pick up the basics which are all you need to get started.

Installing

Getting scss up and running couldn’t be simpler, either using the command line [1] (if your feeling brave), or use a pre-processing app, free or paid (low cost and worth it). Check this page for installing, available apps and using scss options.

Pre-Processors

Two examples of preprocessors are CodeKit (Mac only) and Preprose (Mac and Windows). Both are relatively cheap and good at what they do. They do other stuff besides processing scss to css, they compile all the cutting-edge languages like Less, Stylus and coffeeScript. They live-refresh your browsers. They combine, minify and syntax-check javaScript and crunch your images. Well worth their price.

Partials

The great thing about scss is the ability to create partial files and gather them via a master stylesheet. The partials can be collected in folders for better organisation and easier navigation/access. Partials begin their name with an underscore(_).

@Import

The partial files are collected by using the @import option. Any stylesheet can import partial stylesheets but the end/master stylesheet has no underscore(_) in the name. This stylesheet will be parsed by the preprocessor, a typical master stylesheet is style.scss which will be parsed as style.css.

Variables

Variables speed up your website development and are one of the best reasons for using scss. The variables stylesheet is where you set all the site basics, typeface, font-size, colors, grid settings, etc. You can completely change the look of your site from this stylesheet.

Scenario, you have a 100 page website with a corporate colour of blue, used in the headers as a background colour for the company logo, the colour is also used for some text colour in the body on every page. The boss changes the corporate colour to orange and wants this change implemented on his website. Using css the developer would have to manually change all the instances where the corporate colour occurs on all 100 pages. Using scss the developer would have set the corporate colour as a variable $corporate-color: blue; All the developer has to do is change the variable, $corporate-color: orange; Save the scss file and every instance of $corporate-color; is changed to orange.

How long did that take? The power of the variable is one of the best things about scss.

// The variables stylesheet, you can completely change the look of a website from this stylesheet

$midgrey: #738a94;  
$blue: #3eb0ef;  
$red: #f05230;  
$border: 1px solid tint($red, 20%);  
$radius: 6px;  
$spacer: 4rem;  
$width: $spacer/1.5;

Mixins

Mixins [2] are like shortcuts for writing repeat blocks of code and like variables help speed up website development.

This background mixin sets a background colour with a box shadow using the same colour but 25% darker.

//@include background($red); or @include background();default $blue

@mixin background($background-color: $blue) {  
 background: $background-color;  
 box-shadow: inset 0 0 4px darken($background-color, 25%);  
}

We call this mixin using @include background(value);

The default background colour is $blue, $background-color can be set to any colour - example @include background($red); or @include background(#fecd35);

@include background(); No value set between the parenthesis sets the default color $blue.

The following mixin does not have a default radius size so one must be set every time - @include border-radius(10px);

//@include border-radius(10px);no default radius

@mixin border-radius($radius) {  
    border-top-left-radius: $radius;  
    border-top-right-radius: $radius * 2;  
    border-bottom-left-radius: $radius + 15;  
    border-bottom-right-radius: $radius + 25;  
}

Example, the two above Mixins. See the variables section at the top of the article for the variables in the example.

.box-scss {  
    width: 80%;  
    height: 100px;  
    padding: $spacer;//see the variables at the top of the article  
    margin: $width auto;  
    @include background($red);  
    @include border-radius(10px);

    p {  
    text-align: center;  
    color: white;  
    }
}

Output CSS

.box-scss {  
    width: 80%;  
    height: 100px;  
    padding: 4rem;  
    margin: 2.6666666667rem auto;  
    background: #3eb0ef;  
    -webkit-box-shadow: inset 0 0 4px #0d6ca0;  
    box-shadow: inset 0 0 4px #0d6ca0;  
    border-top-left-radius: 10px;  
    border-top-right-radius: 20px;  
    border-bottom-left-radius: 25px;  
    border-bottom-right-radius: 35px;  
}

.box-scss p {  
    text-align: center;  
    color: white;  
}

Nesting

Scss allows you to nest selectors like HTML.

.box2-scss {  
    border: $border;//see the variables at the top of the article  
    @include background($midgrey);  
    max-width: 80%;  
    padding: $spacer;  
    margin: $spacer auto;

        h1 {  
            font-size: 32px;  
            text-align: center;

            a {  
                color: $blue;

                &:hover {  
                    color: white;  
                }// end hover  
            }// end a  
        }// end h1

        p {  
        text-align: center;  
        color: white;  
        }// end p  
    }// end .div

Note: Too much nesting can become complicated

Output css - see the variables in the variables section at the top of the article

.box2-scss {  
    border: 1px solid #cbd7dd;  
    background-color: #738a94;  
    box-shadow: inset 0 0 4px #3b474d;  
    max-width: 80%;  
    padding: 4rem;  
    margin: 4rem auto;  
}

.box2-scss h1 {  
    font-size: 32px;  
    text-align: center;  
}

.box2-scss h1 a {  
    color: #3eb0ef;  
}

Result

Example of Nesting

@Extend

Another very useful feature of scss is @extend. This feature allows the properties of the class .box3-scss to be shared with the classes .box4-scss and box5-scss. Check the very neat output of the @extend feature.

.box3-scss {  
    width: 80%;  
    margin: $width auto;//see the variables at the top of the article  
    border: $border;  
    @include border-radius(10px);  
    padding: $spacer/2;  
    background-color: $midgrey;

    p {  
    text-align: center;  
    color: white;  
    }
}

.box4-scss {  
    @extend .box3-scss;  
    background-color: $blue;  
}

.box5-scss {  
    @extend .box3-scss;  
    background-color: $red;  
}

Output css

.box3-scss, .box4-scss, .box5-scss {  
    width: 80%;  
    margin: 2.6666666667rem auto;  
    border: 1px solid#cbd7dd;  
    border-top-left-radius: 10px;  
    border-top-right-radius: 20px;  
    border-bottom-left-radius: 25px;  
    border-bottom-right-radius: 35px;  
    padding: 2rem;  
    background-color: #738a94;  
}

.box3-scss p, .box4-scss p, .box5-scss p {  
text-align: center;  
color: white;  
}

.box4-scss {  
    background-color: #3eb0ef;  
}

.box5-scss {  
    background-color: #f05230;  
}

Operators

Scss allows the use of math operators like + - * / and %.

.box6-scss {  
    $width: 300px + 10px;// variable only applies to this class  
    width: $width;  
    margin: $width / 9 - 10px auto;  
    border: $border;  
    padding: $spacer;// see variables section at the top of the article

    p {  
        font-size: $width / 9;  
        text-align: center;  
        line-height: 1.2;  
    }
}

Note: When a variable is set inside a class, it only applies to that class.

Output css

.box6-scss {  
    width: 310px;  
    margin: 89.6666666667px auto;  
    background: #cbd7dd;  
    padding: 4rem;  
}

.box6 p {  
    font-size: 36.5555555556px;  
    text-align: center;  
}

Result

When a variable is set inside a class, it only applies to that class

And Finally

As you can see the basics are easy to pick up. So do yourself a favour and start using scss, a smart way to style your web pages. See the Sass website for more information and examples of scss.

  1. The Command Line can be quite daunting if your not used to it.  ↩

  2. The examples have been run through Autoprefixer using CodeKit.
    Autoprefixer?
    Autoprefixer parses CSS files and adds vendor prefixes to CSS rules using the Can I Use database to determine which prefixes are needed.
    Instead of being a preprocessor – such as Sass and Stylus – Autoprefixer is a postprocessor. It doesn’t use any specific syntax and works with common CSS. Autoprefixer can be easily integrated with Sass and Stylus, since it runs after CSS is already compiled.  ↩
